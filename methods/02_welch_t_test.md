# Критерий Уэлча

## Что это такое

Критерий Уэлча — это вариант `t`-теста для сравнения средних двух независимых выборок.

Он проверяет гипотезу:

- `H0`: средние двух групп равны
- `H1`: средние различаются

## Почему использовался именно он

Обычный двухвыборочный `t`-тест предполагает равенство дисперсий.

Критерий Уэлча лучше подходит, когда:

- размеры групп разные;
- дисперсии групп могут различаться.

В нашем проекте это как раз так:

- группа `Attack` намного больше группы `Normal`;
- разброс значений тоже различается.

Поэтому критерий Уэлча здесь корректнее обычного `t`-теста.

## Что сравнивали в проекте

Сравнивался признак:

- `payload_bytes_per_second`

между двумя группами:

- `Attack`
- `Normal`

## Как это считалось в проекте

В коде тест запускается в функции `run_hypothesis_tests()` через:

```python
attack_values = df.loc[df["is_attack"] == 1, TTEST_FEATURE]
normal_values = df.loc[df["is_attack"] == 0, TTEST_FEATURE]

stats.ttest_ind(
    attack_values,
    normal_values,
    equal_var=False,
    nan_policy="omit",
)
```

Ключевой момент:

- `equal_var=False` означает, что используется именно вариант Уэлча.

В нашем проекте:

- `TTEST_FEATURE = "payload_bytes_per_second"`
- `attack_values` — это все значения `payload_bytes_per_second` для атакующего трафика;
- `normal_values` — это все значения `payload_bytes_per_second` для нормального трафика.

То есть по коду мы буквально разделили столбец `payload_bytes_per_second` на две группы и подали их в тест.

Если смотреть по шагам:

1. Из датафрейма выбираются строки, где `is_attack == 1`.
2. Из них берется признак `payload_bytes_per_second`.
3. Аналогично выбираются строки, где `is_attack == 0`.
4. Обе выборки сравниваются функцией `stats.ttest_ind(..., equal_var=False)`.
5. На выходе получаются `t_stat` и `t_pvalue`.

В коде это находится в `src/analyze_rt_iot2022.py`, строки около `192-199`.

## Что возвращает тест

Тест возвращает:

- `t-statistic`
- `p-value`

Если `p-value < 0.05`, нулевая гипотеза отвергается.

## Что получилось в проекте

- `p-value = 0.0` в машинном выводе

Это означает, что значение настолько мало, что гипотеза о равенстве средних уверенно отвергается.

Следовательно, средние `payload_bytes_per_second` у `Attack` и `Normal` различаются статистически значимо.

## Что такое размер эффекта

После критерия Уэлча в проекте дополнительно считался размер эффекта `Cohen's d`.

Это нужно потому, что:

- `p-value` показывает, есть ли статистически значимое отличие;
- `Cohen's d` показывает, насколько это отличие велико по сути.

## Как считался Cohen's d

Сначала считалось объединенное стандартное отклонение:

`pooled_std`

`pooled_std` — это объединенное стандартное отклонение двух групп.

Проще говоря, это общий масштаб разброса значений в двух сравниваемых выборках.

Он нужен затем, чтобы разность средних измерять не просто в исходных единицах, а в единицах типичного разброса данных.

Именно поэтому `Cohen's d` удобно интерпретировать:

- если разность средних мала по сравнению с разбросом, эффект маленький;
- если разность средних велика по сравнению с разбросом, эффект большой.

затем:

`Cohen's d = (mean_attack - mean_normal) / pooled_std`

В коде это реализовано так:

```python
pooled_std = np.sqrt(
    (
        attack_values.var(ddof=1) * (len(attack_values) - 1)
        + normal_values.var(ddof=1) * (len(normal_values) - 1)
    )
    / (len(attack_values) + len(normal_values) - 2)
)
cohens_d = (attack_values.mean() - normal_values.mean()) / pooled_std
```

То есть:

1. Сначала считается объединенное стандартное отклонение двух групп.
2. Затем разность средних делится на этот общий масштаб разброса.

Если говорить совсем просто, `pooled_std` отвечает на вопрос:

`насколько в среднем сильно "шумят" значения внутри групп`

а `Cohen's d` показывает:

`насколько далеко средние находятся друг от друга по сравнению с этим шумом`

## Как интерпретировать Cohen's d

Обычно:

- `0.2` — маленький эффект
- `0.5` — средний
- `0.8` и выше — большой

В проекте:

- `Cohen's d = 1.07`

Это большой эффект.

То есть различие между группами не только статистически значимо, но и существенно по величине.
