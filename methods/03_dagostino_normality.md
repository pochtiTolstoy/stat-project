# Тест Д'Агостино на нормальность

## Что это такое

Тест Д'Агостино, или `D'Agostino K^2 test`, — это статистический тест на нормальность распределения.

Он проверяет гипотезу:

- `H0`: данные имеют нормальное распределение
- `H1`: данные не имеют нормального распределения

## На чем основан тест

Тест использует две характеристики распределения:

- асимметрию
- эксцесс

Если распределение сильно отличается от нормального по этим характеристикам, тест дает малое `p-value`, и гипотеза о нормальности отвергается.

## Почему он подходит для проекта

- по ТЗ нужно было проверить гипотезу о виде распределения;
- это классический тест на нормальность;
- он хорошо подходит для достаточно больших выборок.

## Почему проверяли только нормальный трафик

В проекте тест применялся к `flow_duration` только для класса `Normal`.

Причина такая:

- нормальный трафик отражает штатное поведение системы;
- атакующий трафик состоит из смеси разных типов атак;
- смесь разных атак сама по себе может давать сложное распределение, которое труднее интерпретировать.

Поэтому проверка нормальности на нормальном трафике содержательнее.

## Как это считалось в проекте

В коде:

1. Брались значения `flow_duration` только для нормального трафика.
2. Из них выбиралась выборка до `5000` наблюдений.
3. Затем вызывался тест:

```python
normality_sample = (
    df.loc[df["traffic_class"] == "Normal", NORMALITY_FEATURE]
    .sample(
        n=min(TEST_SAMPLE_SIZE, (df["traffic_class"] == "Normal").sum()),
        random_state=RANDOM_STATE,
    )
    .astype(float)
)

k2_stat, normality_pvalue = stats.normaltest(normality_sample)
```

Разберем этот код по частям.

### 1. Отбор только нормального трафика

```python
df.loc[df["traffic_class"] == "Normal", NORMALITY_FEATURE]
```

Здесь происходит следующее:

- `df["traffic_class"] == "Normal"` создает логическую маску;
- она отмечает только те строки, где трафик относится к классу `Normal`;
- `NORMALITY_FEATURE` в проекте равен `"flow_duration"`.

Итог: из всей таблицы выбирается один столбец `flow_duration`, но только для нормального трафика.

То есть по смыслу это:

`возьми значения flow_duration только там, где трафик нормальный`

### 2. Случайная выборка из этих значений

```python
.sample(
    n=min(TEST_SAMPLE_SIZE, (df["traffic_class"] == "Normal").sum()),
    random_state=RANDOM_STATE,
)
```

Здесь выбирается случайная подвыборка.

Что означает параметр `n`:

- `TEST_SAMPLE_SIZE` в проекте равно `5000`;
- `(df["traffic_class"] == "Normal").sum()` — это число строк с нормальным трафиком;
- `min(...)` берет меньшее из этих двух чисел.

Значит логика такая:

- если нормальных наблюдений больше `5000`, берем `5000`;
- если их меньше `5000`, берем все.

Почему это удобно:

- тест проводится на достаточно большой выборке;
- вычисления остаются стабильными и быстрыми;
- не приходится гонять тест по всему массиву, если он очень большой.

Параметр:

```python
random_state=RANDOM_STATE
```

нужен для воспроизводимости.

Это означает, что случайная выборка каждый раз будет одна и та же, если код запускать снова с тем же `random_state`.

### 3. Приведение к числовому типу

```python
.astype(float)
```

Это приводит значения к типу `float`.

Зачем это нужно:

- статистические функции ожидают числовые данные;
- так мы явно гарантируем правильный формат входа для теста.

### 4. Что такое normality_sample

После этого блока переменная `normality_sample` содержит:

- одномерную выборку чисел;
- это значения `flow_duration`;
- только для нормального трафика;
- размером до `5000`;
- готовые для применения статистического теста.

### 5. Запуск самого теста

```python
k2_stat, normality_pvalue = stats.normaltest(normality_sample)
```

Здесь вызывается готовая функция `normaltest` из `scipy.stats`.

Она реализует тест Д'Агостино на нормальность.

На вход подается выборка `normality_sample`.

На выходе функция возвращает два значения:

- `k2_stat` — статистика теста;
- `normality_pvalue` — `p-value`.

### 6. Как интерпретировать результат

Дальше используется обычное правило:

- если `p-value < 0.05`, гипотеза о нормальности отвергается;
- если `p-value >= 0.05`, оснований отвергать нормальность нет.

В нашем проекте `p-value` получилось чрезвычайно малым, поэтому гипотеза о нормальности была отвергнута.

В нашем проекте:

- `NORMALITY_FEATURE = "flow_duration"`
- `TEST_SAMPLE_SIZE = 5000`

То есть по коду мы сделали следующее:

1. Взяли только строки, где `traffic_class == "Normal"`.
2. Из них выбрали столбец `flow_duration`.
3. Случайно взяли из него выборку размером до `5000` наблюдений.
4. Преобразовали значения в `float`.
5. Передали эту выборку в `stats.normaltest(...)`.

На выходе:

- `k2_stat` — статистика теста Д'Агостино;
- `normality_pvalue` — `p-value`.

В коде это находится в `src/analyze_rt_iot2022.py`, строки около `210-215`.

## Почему брали выборку из 5000

Это сделано для устойчивого и удобного вычисления на большом датасете.

Сам тест все равно применяется к достаточно большой выборке, так что для вывода этого достаточно.

## Что получилось в проекте

- статистика теста: `14454.66`
- `p-value = 0.0` в машинном выводе

Это не буквальный математический ноль.  
На самом деле `p-value` просто чрезвычайно мало, порядка `10^-3139`, поэтому стандартный `float` уже выводит его как `0.0`.

Следовательно, гипотеза о нормальности отвергается.

## Как это интерпретировать

Это не ошибка и не проблема.

Для сетевого трафика нормальность часто не выполняется, потому что:

- распределения асимметричны;
- есть длинные хвосты;
- есть выбросы;
- встречаются резкие всплески нагрузки.
